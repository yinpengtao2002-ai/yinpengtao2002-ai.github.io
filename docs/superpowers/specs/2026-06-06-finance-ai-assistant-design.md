# 财务分析 AI 助手设计

## 背景

站内已有全局 AI 助手，但它只负责网站内容导览、财务模型说明和页面推荐，不读取用户上传数据。新工具应作为独立页面存在，用于用户主动上传经营明细后，通过持续聊天获得精确问答、图表和财务分析结论。

第一版先做窄：只支持一张通用经营明细表，不做数据库、不做跨文件、不做自由 BI、不做比率指标体系。目标是把“用户提问 -> AI 选择分析动作 -> 前端确定性计算 -> 聊天消息内展示图表和口径”跑通。

## 目标

- 用户上传 CSV、XLSX 或 XLS 经营明细后即可提问，不需要先配置字段。
- 原始底稿只保存在浏览器当前页面内存中，刷新后清空。
- AI 可以持续聊天，但不直接计算数字，也不自由读取完整底稿。
- 每轮 AI 回复可以包含 1 到 3 个图表或指标模块，并且这些模块必须作为聊天回复的一部分显示在聊天流里。
- 支持精确问答，例如“今年 3 月巴西的单车边际是多少？环比多少？同比多少？”
- 支持分析问答，例如“3 月利润下降主要来自哪些国家？”
- 图表第一版只支持趋势图、横向柱状排名图和瀑布桥图。

## 非目标

- 不接入全局站内 AI 助手。
- 不保存用户上传数据到数据库。
- 不在服务端持久化数据集。
- 不支持多文件、多 sheet 合并或跨刷新恢复。
- 不支持毛利率、边际率、成本收入比等比率指标的自动可信计算。
- 不做复杂销量结构/单车质量归因，瀑布桥第一版只按维度拆变化贡献。
- 不做字段确认向导，除非关键字段缺失或明显冲突。

## 产品入口

建议路由：

- 页面：`/tools/finance-ai-assistant`
- API：`/api/tools/finance-ai-assistant`

该工具遵循站内独立 AI 工具模式：页面与全局 AI 助手隔离，但可以复用现有 AI provider 配置、API key 读取方式和 fallback provider 机制。

## 页面体验

页面是一个聊天式分析工作台，而不是传统 dashboard。图表必须嵌入 AI 消息，作为正常聊天回复的一部分出现。

基本布局：

```text
顶部：工具名称、数据状态、轻量提示
上传区：上传 CSV / XLSX / XLS，显示识别摘要
主区：持续聊天流
  - 用户消息
  - AI 文本结论
  - AI 图表卡片 1 到 3 个
  - 口径说明和命中行数
底部：输入框
```

上传成功后的识别摘要保持轻量：

```text
已识别：20,480 行 / 18 个月 / 5 个维度 / 4 个总额指标 / 4 个单车指标
```

不展示字段确认表单。只有关键字段缺失时才打断用户。

## 数据流

```text
上传 Excel/CSV
-> 前端解析完整底稿
-> 前端自动生成字段画像和指标 schema
-> 用户持续聊天提问
-> API 基于字段画像、最近状态和用户问题返回结构化动作
-> 前端校验动作
-> 前端用完整底稿确定性计算
-> 聊天消息中展示趋势图 / 横向柱状图 / 瀑布桥
-> API 基于计算结果摘要生成最终解释
```

关键边界：

- 完整底稿只在浏览器内存中。
- API 不接收完整明细表。
- AI 不做数值计算，不决定最终口径。
- 前端分析执行层负责筛选、聚合、环比、同比、单车指标和图表数据。

## 字段识别

第一版只支持通用经营明细表：

```text
月份 + 维度列 + 销量 + 总额指标列
```

示例：

```text
月份 / 大区 / 国家 / 车型 / 渠道 / 销量 / 净收入 / 成本 / 边际 / 利润
```

字段识别规则沿用现有财务模型习惯：

- 月份列：识别 `月份`、`年月`、`期间`、`月度`、`month`、`date`、`period` 等别名。
- 销量列：识别 `销量`、`销售量`、`发车量`、`台数`、`数量`、`volume`、`qty`、`units` 等别名。
- 维度列：默认是销量列之前且不是月份列的字段。
- 总额指标列：默认是销量列之后的数值字段。
- 非默认指标：字段名含 `率`、`占比`、`%`、`单车`、`单位`、`平均`、`均价`、`单价` 的列第一版不主动用于默认分析。

关键字段要求：

- 必须能识别月份列。
- 必须能识别销量列。
- 必须至少有一个销量列之后的总额指标。

## 指标口径

第一版核心指标分两类：

```text
可加总指标：销量、净收入、成本、边际、利润、费用等
单车派生指标：总额指标 / 销量
```

单车指标示例：

- 单车净收入 = sum(净收入) / sum(销量)
- 单车成本 = sum(成本) / sum(销量)
- 单车边际 = sum(边际) / sum(销量)
- 单车利润 = sum(利润) / sum(销量)

单车指标必须在目标分析粒度上先聚合再计算，不能逐行相除后平均。

示例：

```text
国家=巴西，2026-03

行1：销量 100，边际 3000
行2：销量 0，边际 500

单车边际 = (3000 + 500) / (100 + 0) = 35
```

零销量金额行规则：

- 零销量但有金额的行不能丢弃。
- 只要月份和维度匹配，金额必须进入总额聚合。
- 单车分母只使用聚合后的销量合计。
- 聚合后销量为 0 时，总额指标仍展示，单车指标返回不可计算。
- 趋势图、横向柱状图、瀑布桥和精确问答必须共享同一口径。

环比同比：

- 环比 = 当前月对上月。
- 同比 = 当前月对去年同月。
- 缺少可比月份时，展示当前值并明确说明缺少上月或去年同月。

## AI 动作协议

AI 第一轮负责把用户问题转成结构化动作。每轮最多返回 1 到 3 个模块。

支持动作：

- `metric_snapshot`：精确指标问答。
- `trend_chart`：趋势图。
- `bar_rank`：横向柱状排名图。
- `waterfall_bridge`：瀑布桥图。

AI 返回的字段、指标、维度、筛选、月份和图表类型必须经过前端 schema 校验。非法字段或非法动作不得执行。

精确问答示例：

```json
{
  "modules": [
    {
      "type": "metric_snapshot",
      "metric": "单车边际",
      "period": "2026-03",
      "filters": { "国家": ["巴西"] },
      "comparisons": ["mom", "yoy"],
      "chart": {
        "type": "trend_chart",
        "highlightPeriod": "2026-03"
      }
    }
  ]
}
```

分析问答示例：

```json
{
  "modules": [
    {
      "type": "trend_chart",
      "metric": "利润",
      "filters": {},
      "focusPeriod": "2026-03"
    },
    {
      "type": "bar_rank",
      "metric": "利润",
      "dimension": "国家",
      "period": "2026-03",
      "comparison": "mom",
      "sort": "change_asc",
      "limit": 10
    },
    {
      "type": "waterfall_bridge",
      "metric": "利润",
      "dimension": "国家",
      "fromPeriod": "2026-02",
      "toPeriod": "2026-03",
      "limit": 8
    }
  ]
}
```

## 图表层

图表第一版只做三类，全部作为 AI 聊天消息卡片渲染。

### 趋势图

用途：

- 展示某指标随月份变化。
- 支持总额指标和单车派生指标。
- 精确问答场景下可高亮目标月份。

口径：

- 总额指标按月求和。
- 单车指标按月 `sum(metric) / sum(销量)`。

### 横向柱状图

用途：

- 展示 Top/Bottom 排名。
- 展示某维度变化贡献或某期间表现排名。

规则：

- 只做横向柱状图。
- 不在图旁额外列数字表。
- 柱尾可显示数值，hover 可显示口径、筛选和命中行数。

### 瀑布桥图

用途：

- 展示从 A 月到 B 月的指标变化。
- 第一版按单个维度拆变化贡献，例如国家、车型或渠道。

规则：

- 起点是 fromPeriod 指标值。
- 中间项是维度变化贡献。
- 终点是 toPeriod 指标值。
- Top-N 之外的维度合并为其他。

## 聊天上下文与缓存

为了提高缓存命中和响应稳定性，prompt 分三块：

1. 固定系统提示：角色、动作白名单、图表模板、输出 JSON、AI 不算数等规则。
2. 稳定数据画像：上传后生成一次，包含字段字典、维度/指标、数据规模、可用月份、可用派生指标。
3. 轻量聊天状态：最近几轮问题、已生成 chartSpec、当前指标、当前筛选。

完整底稿不进入聊天上下文。图表明细结果也不长期塞入上下文，只传必要计算摘要给 AI 生成解释。

## API 职责

`/api/tools/finance-ai-assistant` 承担两类调用：

- `plan`：根据用户问题、字段画像和最近状态返回结构化动作。
- `explain`：根据前端计算结果摘要生成简短解释。

API 不保存数据库，不接收完整底稿，不返回裸模型名给用户。它可以复用现有 DeepSeek + OpenAI-compatible fallback provider 链。

## 异常处理

- 未上传数据：提示先上传经营明细。
- 找不到月份：提示需要月份、年月或期间列。
- 找不到销量：提示需要销量列，因为单车指标和部分口径依赖销量。
- 找不到总额指标：提示销量列之后需要至少一个数值指标。
- 用户问不存在的字段：明确说明当前底稿没有该字段，并推荐可用指标。
- 用户问同比但缺少去年同月：展示当前值和环比，说明缺少去年同月。
- 聚合后销量为 0：总额照常展示，单车指标显示不可计算。
- AI 返回非法字段或动作：前端拒绝执行，并请求 API 基于可用字段重新规划。

## 测试范围

- 上传字段识别：销量列之前识别为维度，销量列之后识别为总额指标。
- 单车指标：必须按聚合后 `sum(metric) / sum(销量)` 计算。
- 零销量金额行：金额参与总额聚合，单车分母不变。
- 环比同比：正确定位上月和去年同月。
- 精确问答动作：巴西 3 月单车边际返回当前值、环比、同比和趋势图 spec。
- 分析问答动作：利润下降问题可生成趋势图、横向柱状排名和瀑布桥。
- 动作校验：非法字段、非法动作和超过 3 个模块不得执行。
- 聊天渲染：图表卡片必须出现在 AI 消息内，而不是独立图表区。

## 第一版成功标准

- 用户上传一张经营明细表后，无需字段配置即可开始提问。
- 用户问“今年 3 月巴西单车边际是多少，环比同比多少”，工具能返回准确数值、环比、同比、趋势图和口径说明。
- 用户问“3 月利润下降主要来自哪些国家”，工具能返回趋势图、横向柱状排名、瀑布桥和简短解释。
- 零销量但有金额的行在总额和单车口径中被正确处理。
- 图表作为 AI 聊天回复的一部分渲染。
- 刷新页面后上传数据清空。
- 全站 AI 助手仍然不能读取上传底稿。

## 后续扩展

- 公式指标：支持毛利率、边际率、成本收入比等 `A / B` 口径。
- 字段修正：在识别出错时提供轻量手动修改，而不是完整配置向导。
- 更强瀑布桥：引入销量结构、单车质量和总额变化的复合归因。
- 服务端临时数据会话：在未来有存储或队列能力时再考虑。
- 多 sheet 和多文件：在第一版稳定后再做。
