# Lucas Yin 网站全站重构设计

日期：2026-05-05

## 1. 定位

新网站不是传统作品集，也不是简历时间线，而是一份能力和思考展示。

核心判断：Lucas 从业务问题出发，持续打磨经营分析、财务模型与 AI 工作流。

主读者是未来老板、招聘方和猎头。次读者是当前同事、跨部门合作方，以及可能使用财务模型的内部伙伴。

表达原则：

- 不写“我在找机会”。
- 不做经历时间线。
- 不堆技能标签。
- 不把公司名放首屏，但 footer 可以低调保留当前身份。
- 用真实模型、工具、文章和 AI 助手证明能力。
- 整体像高级能力档案加工具平台，保留 Claude 式温暖色系和字体气质。

## 2. 首页

首页保留大屏段落感，但每屏信息密度提高。移动端逻辑与桌面完全一致，只做更轻的排布。

### 首屏

左侧保留 `Lucas Yin` 的视觉记忆点，但压缩面积。

左侧身份线：

```text
经营分析 · 财务模型 · AI 工作流
```

右侧主标题：

```text
从业务问题出发，持续打磨经营分析、财务模型与 AI 工作流。
```

首页不写“这不是经历列表”这类解释性文案，让结构自己说明定位。

首屏三条能力证据：

- 财务模型
- AI 工作流
- 思考判断

每条证据需要包含一句能力说明，并挂一个代表入口。说明文案偏直接能力型，不写成自夸口号。

首屏只保留一个主按钮：

```text
查看财务模型
```

### 导航

最终主导航：

```text
首页 / 财务模型 / 思考与方法 / 联系
```

对应路径：

- `/`
- `/finance`
- `/thinking-lab`
- `/#contact`

桌面端保留顶部胶囊导航。移动端保留轻量目录按钮。不做 `/about`，不做能力档案页面。

### 后续屏幕

第二屏是财务模型区，默认展示全部分类和全部模型。

第三屏是思考与方法，展示主题入口和最新内容。

末尾是联系区和当前身份，保留邮箱、LinkedIn、微信，以及低调的当前工作身份。

## 3. 财务模型库

`/finance` 从文章列表改成模型工作台和模型库。首页财务区是精简版，`/finance` 是完整版，两者共享同一套本地模型 registry。

原则：

- 财务模型仍然由本地 repo 维护，不走 Notion。
- 模型本体、示例数据、分类、入口和 AI 说明都在 repo 中。
- 模型卡片保持轻量，只放标题、一句话简介和入口。
- 点击模型卡片直接进入工具本体。
- 工具页默认自动加载示例数据，让用户一打开就能看到模型效果。
- 工具页保留上传入口，方便真实业务使用。
- 未来新增模型时，只新增模型配置和路由，不重写页面结构。

初始分类：

- 预算复盘：预算实际对比模型
- 趋势监控：分月指标趋势分析模型
- 利润模拟：利润敏感性分析
- 单车归因：单车指标变动归因模型

财务模型页必须继续保留现有 full-screen tool 的使用习惯，包括移动端控制台、返回入口和既有工具交互一致性。

## 4. 思考与方法

`AI 工作流` 和 `日常随笔` 合并为 `思考与方法`。

导航中文：`思考与方法`

新路径：`/thinking-lab`

内容源：

- 思考与方法文章全部由 Notion 负责。
- 先兼容已有 Notion 数据结构，不要求立即整理 Notion。
- 网站生成时合并现有 AI 和 essays 内容。
- 分类标签从 Notion 现有字段读取，不在代码里写死。
- 没有分类时使用兜底分类，例如“思考记录”。

页面结构：

- `/thinking-lab` 顶部是主题地图。
- 下方是持续更新的文章流。
- 支持按 Notion 分类筛选。
- 文章详情统一为 `/thinking-lab/<slug>`。
- 文章详情页保持当前清晰阅读风格：标题、日期、正文。

旧链接策略：

- `/ai` 永久跳转到 `/thinking-lab`。
- `/essays` 永久跳转到 `/thinking-lab`。
- `/article/ai/<slug>` 永久跳转到 `/thinking-lab/<slug>`。
- `/article/essays/<slug>` 永久跳转到 `/thinking-lab/<slug>`。
- 删除旧 `/article/[category]/[slug]` 渲染页，用 Next.js redirects 处理。
- sitemap 只输出新路径。

财务模型路径保持 `/finance/<slug>` 不变。

## 5. AI 助手

右下角 AI 助手保留，并作为网站重点亮点之一，但入口保持克制。

入口：

- 保留右下角浮窗按钮。
- 不放进导航。
- 不在首屏抢主视觉。
- 移动端保留，但不能遮挡内容或影响阅读。

能力升级：

- 能介绍 Lucas 的网站结构。
- 能推荐合适的财务模型。
- 能解释每个模型怎么用。
- 能根据当前模型页回答这个模型适合什么场景、怎么开始。
- 能推荐 `/thinking-lab` 里的相关文章。
- 能解释 Lucas 的能力结构，但语气不自夸、不像客服。

数据来源：

- 财务模型读取本地模型 registry 和模型说明数据。
- 思考与方法读取 Notion 生成内容。
- 模型说明数据包括用途、适合场景、使用步骤、示例数据说明、常见问题。
- 初始模型说明数据由 AI 根据现有工具页面、模型文案和示例数据生成，写入 repo。
- 发布后说明数据保持稳定，可人工修改；不让运行时 AI 动态乱编。

语气：

- 面向访客。
- 不暴露内部索引、fallback、生成逻辑。
- 不写“站内索引”这类实现语言。
- 更像懂这个网站的向导。

## 6. 清理范围

本次按一次性大工程做，采用整站垂直重构。

用户路径需要一次打通：

```text
首页 → 财务模型库 → 工具页示例数据 → thinking-lab → 文章页 → AI 助手 → redirects → sitemap
```

删除或下线：

- 删除旧 `/explore`。
- 删除不再服务新结构的旧组件。
- 删除旧 `/article/[category]/[slug]` 渲染页。
- `/ai`、`/essays` 只保留 redirect，不再作为主页面。
- 不做 `/about`。
- 不做传统简历时间线。
- 不堆技能标签。

必须保留：

- 现有财务工具本体和 URL：`/finance/<slug>`。
- 财务工具的移动端控制台习惯。
- 工具页返回、控制台等既有一致性。
- 文章 Markdown 阅读体验。
- Claude 式温暖色系和字体气质。
- footer 中邮箱、LinkedIn、微信，以及低调当前身份。

## 7. 验证

实施完成后需要验证：

- `npm run lint`
- `npx tsc --noEmit`
- `npm run build:vercel`
- 财务模型相关测试
- 桌面和移动端视觉检查
- redirects 检查
- sitemap 检查
- AI 助手基础问答检查

财务模型页面至少覆盖桌面和移动端关键路径。移动端重点检查模型页控制台打开、关闭、返回、示例数据可见性和内容不被 AI 助手遮挡。

## 8. 实施策略

采用整站垂直重构，不切 MVP。

推荐执行顺序：

1. 建立内容和模型 registry。
2. 重构路由、redirects 和 sitemap。
3. 重做首页。
4. 重做 `/finance`。
5. 新建 `/thinking-lab` 和文章详情页。
6. 扩充 AI 助手知识数据和回答逻辑。
7. 清理旧页面和旧组件。
8. 统一视觉 polish。
9. 完整验证并修复发现的问题。

每个阶段都必须围绕同一条主线：让访客快速判断 Lucas 的能力，并能立即打开真实产出。
