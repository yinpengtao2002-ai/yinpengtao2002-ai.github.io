> 来源：`docs/website-improvement-questions.md` 总清单中的前 3 条。后续建议仍留在总清单里，暂不进入本轮修改。

## 本轮激活范围

1. 首页能力主线收窄为：`业务问题 -> 财务模型 -> 分析判断 -> 可视化表达`。
2. Hero 首屏加入更吸引人的“业务问题 -> 财务模型”入口。
3. AI 助手升级为模型使用、口径校对、可视化建议和分析框架助手。

## 明确不动

- 不动首页第二页财务模型展示。
- 不改 `/finance` 模型列表页结构。
- 不改具体财务模型内页结构。
- 不把 AI 工作流和思考方法硬塞进财务能力主线。
- 不把“AI 不读取数据、不分析数据”作为前台卖点反复展示。

## 1. 首页能力主线

### 决策

Hero 和首页前半段重点表达财务分析能力链路：

```text
业务问题 -> 财务模型 -> 分析判断 -> 可视化表达
```

AI 工作流、思考方法仍然是网站能力的一部分，但作为独立入口出现，不参与这条主链路。

### 目标表达

用户进入网站后，应快速感知：

- Lucas 关注真实经营问题，而不是只展示工具。
- 财务模型用于把问题结构化。
- 图表和可视化用于帮助别人理解判断。

### 建议文案

```text
从经营问题到模型、判断和图表
```

```text
把预算偏差、利润变化、趋势异常和单车指标拆成可计算的模型，
再用图表和判断把结果讲清楚。
```

## 2. Hero 首屏业务问题入口

### 决策

只在最上面的 Hero 首屏加入业务问题入口。首页第二页财务模型区继续保留现在的模型名称和现有展示方式。

### 初版问题入口

| 业务问题 | 对应模型 |
| --- | --- |
| 如何找出单车变动的罪魁祸首？ | 使用单车指标变动归因模型 |
| 预算到底偏在哪里？ | 使用预算实际对比模型 |
| 销量、收入、成本哪个最影响利润？ | 使用利润敏感性分析模型 |
| 月度趋势从哪里开始异常？ | 使用分月指标趋势分析模型 |

### 交互

- 桌面端：放在 Hero 右侧产品舞台下方，形成紧凑的问题目录。
- 手机端：显示为紧凑卡片，避免挤压 Lucas 主标题。
- 点击入口：先平滑滚动到第二页财务模型区，不直接改第二页结构。

## 3. AI 助手定位

### 决策

AI 助手定位为：

```text
模型使用助手 + 口径校对助手 + 可视化建议助手 + 分析框架助手
```

它不是“自动数据分析助手”，也不在前台主动强调隐私边界。边界作为内部原则保留：默认不分析用户上传的具体数据。

### AI 应该优先回答

1. 这个模型适合解决什么业务问题？
2. 我应该准备什么格式的数据？
3. 这个模型里的图表应该按什么顺序看？
4. 如果预算和实际偏差很大，我应该先检查哪些口径？
5. 这类问题适合用瀑布图、趋势图、热力图还是敏感性分析？
6. 我想写一段汇报说明，应该怎么组织逻辑？

### 体验要求

- 在模型页面打开 AI 时，展示当前模型相关的推荐问题。
- 回答偏步骤、校对和检查清单，不写成长论文。
- 回答要体现“先看业务问题，再校接口径，再看结构，最后形成判断”的方法。
- 当用户要求分析当前数据时，不假装看到了数据；可以请用户提供截图、关键指标或数据摘要。

## 验收标准

- Hero 首屏不再把主链路写成“模型、图表和 AI 解读”，而是聚焦“模型、判断和图表”。
- Hero 首屏出现业务问题入口，且文案不是功能说明。
- 首页第二页财务模型区保持现有展示方式。
- AI 打开后的欢迎语和推荐问题更像模型使用/校对助手。
- AI 系统提示词不把“不能读取数据”作为前台卖点，但仍禁止假装看到了用户数据。
